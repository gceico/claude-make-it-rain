# CLAUDE.md

## Overview

Make It Rain is an Electron menu-bar/tray app that watches Claude Code usage
logs (`~/.claude/projects/*/*.jsonl`) and shows the day's estimated spend, flying
a 💵 off the tray every whole dollar and raining money at $100 milestones. It can
optionally report an anonymized daily total (a random gamer tag + USD figure) to
a cloud leaderboard. The npm package is `@gceico/claude-make-it-rain` and it
installs a single CLI command, `claude-make-it-rain` (`bin/make-it-rain.js`,
which spawns Electron detached).

The project is written in TypeScript and uses **Bun** as its toolchain
(package manager, test runner, and bundler). Electron itself still runs its
embedded Node, so `bun run build` compiles the TS sources to CommonJS in
`dist/`. Contributors need Bun ≥ 1.3; the _published_ CLI runs under the user's
Node (≥ 22).

## Commands

| Task                              | Command                                                    |
| --------------------------------- | ---------------------------------------------------------- |
| Build (TS → `dist/` + `bin/`)     | `bun run build` (`scripts/build.ts`)                       |
| Run (build + launch Electron)     | `bun start` (alias `bun run dev`)                          |
| Run detached (like installed CLI) | `node bin/make-it-rain.js` (after a build)                 |
| Test everything                   | `bun run test` (runs `test:app` then `test:server`)        |
| Test app only                     | `bun run test:app` (5 suites via `bun test`)               |
| Test server only                  | `bun run test:server` (leaderboard suites, `bun:sqlite`)   |
| Type-check                        | `bunx tsc --noEmit`                                        |
| Lint                              | `bun run lint` (`eslint .`, typescript-eslint flat config) |
| Format                            | `bun run format` / check with `bun run format:check`       |
| Build the web site                | `cd web && bun run build` (Astro → `web/dist`)             |

`bun start`/`bun run dev` build first, then launch Electron. `dist/` is
gitignored; `bin/make-it-rain.js` is a committed build artifact (regenerated by
`bun run build`).

## Architecture

- `src/main.ts` — Electron main process: tray item, overlay window management,
  IPC, single-instance lock, and wiring of the monitor/leaderboard/update-checker.
- `src/bin/make-it-rain.ts` — CLI launcher; spawns Electron (detached by
  default, `--foreground` to stay attached). Built to `bin/make-it-rain.js`.
- `overlay.html` + `src/preload.ts` — transparent, click-through, always-on-top
  fullscreen overlay that renders the dollar-fly and money-rain animations.
  `overlay.html` is hand-tuned and Prettier-ignored (embedded base64 assets).
- `src/lib/` (pure, Electron-free, unit-testable): `usage-monitor.ts`
  (incremental JSONL tailing, dedup, daily reset), `entry-parser.ts`,
  `pricing.ts` (per-model USD/1M-token table), `milestones.ts` ($10/$50 stack
  milestones), `history.ts`, `leaderboard-client.ts` (tag generation, config
  persistence, hourly fail-silent reporting), `update-checker.ts`, `config.ts`,
  `denominations.ts`.
- `scripts/build.ts` — the build: `Bun.build` compiles `src/main.ts`,
  `src/preload.ts` (→ `dist/`) and `src/bin/make-it-rain.ts` (→ `bin/`) to
  standalone CommonJS with `electron` kept external, then injects a node shebang
  and `chmod +x` on the CLI.
- `server/` — zero-dependency **Bun** leaderboard backend: `index.ts`
  (`Bun.serve`; endpoints `POST /api/report`, `GET /api/leaderboard`,
  `GET /api/stars`, `GET /health`, static `/`), `db.ts` (`bun:sqlite` store),
  plus `Dockerfile`, `railway.json`, and a minimal `package.json`
  (`bun index.ts`). Static assets come from `STATIC_DIR` (see gotchas).
- `web/` — Astro 5 static site (TypeScript strict; only dependency is `astro`).
  Ports the old landing page 1:1 (live leaderboard, stars, countdown, money
  rain). `cd web && bun install && bun run build` emits a single fully-inlined
  HTML file + favicon to `web/dist` (CSP-compatible). `web/bun.lock` is committed.
- `test/` — `bun test` (`bun:test`) suites, `*.test.ts`.

## Conventions & invariants

- **TypeScript throughout** (strict mode), CommonJS output for the Electron app,
  ESM/`Bun.serve` for the server, ESM for `web/`. No `'use strict'` pragmas
  needed (TS modules are strict); files carry header doc comments explaining
  intent.
- Prettier config: `singleQuote`, `printWidth: 80`, `trailingComma: "es5"`.
  Lint (`eslint`, typescript-eslint flat config in `eslint.config.js`),
  `bunx tsc --noEmit`, and `format:check` all run in CI.
- `web/.astro`, `web/dist`, and `web/src/styles/global.css` (embedded base64
  font) are Prettier-ignored, as is `overlay.html` and the built
  `bin/make-it-rain.js`.
- Dependency story: the app's only runtime dependency is `electron` (^43);
  `trustedDependencies: ["electron"]` lets Bun run Electron's postinstall.
  `web/`'s only dependency is `astro`. The `server/` backend has **ZERO** npm
  dependencies — it uses `bun:sqlite` and Bun's built-in HTTP (`Bun.serve`).
  Keep it that way.
- `package.json` pins `packageManager: bun@1.3.12` and keeps the `engines.node`
  floor at 22 (the published CLI runs under the user's Node).

## Testing

- Tests are `bun test` files (`import { test } from 'bun:test'`), run via the
  package scripts (no jest/mocha/harness). 56 tests total.
- `test:app` runs the five Electron-free suites (`usage-monitor`, `delight`,
  `leaderboard-client`, `history`, `update-checker`); `test:server` runs the two
  leaderboard suites (`leaderboard-db`, `leaderboard-server`) that use
  `bun:sqlite`.
- Debug/animation hooks are env vars honored by `main.ts` when run via
  `bun start` (NOT used by the automated tests): `MIR_TEST_RAIN=1` (rain after
  1.5s), `MIR_TEST_STACK=<n>` (burst n stacks after 3s), `MIR_TEST_SHOT=/path.png`
  (screenshot the overlay), `MIR_TEST_USER_DATA=/path` (isolated userData dir).

## Gotchas

- Bun is required for development (build/test/tooling); Electron cannot run on
  Bun, so `bun run build` compiles TS to JS that Electron's embedded Node loads.
  Always build before launching (`bun start` does this for you).
- The server uses `bun:sqlite`, not `node:sqlite`. The on-disk SQLite file
  format is unchanged, so the existing production DB (`/data/leaderboard.db`)
  keeps working across the port.
- `STATIC_DIR` env var controls where the server serves the landing page from.
  Resolution when unset: `STATIC_DIR` → `server/public` if it exists →
  `web/dist` (the Astro build output, baked into the Docker image at
  `/app/public`). `server/public` no longer exists in the repo.
- CI sets `ELECTRON_SKIP_BINARY_DOWNLOAD=1` for `bun install` (tests never open
  a window, so the ~100 MB Electron binary is skipped).
- Railway deploys the reference server via a **multi-stage** `server/Dockerfile`
  built with the **repo root** as context (`oven/bun:1-alpine`): stage 1 builds
  `web/` (Astro), stage 2 runs the server with `STATIC_DIR=/app/public`. Both
  the root and `server/` `railway.json` use `builder: DOCKERFILE` with
  `dockerfilePath: server/Dockerfile`. A persistent volume mounts at `/data`
  with `LEADERBOARD_DB=/data/leaderboard.db`. The deploy workflow triggers on
  changes under `server/**`, `web/**`, and `railway.json`.
- Telemetry (leaderboard reporting) is ON by default, opt-out from the tray menu;
  disabling it stops all network activity. The client fails silently if the server
  is unreachable.

See `docs/DECISIONS.md` for deployment/security/anti-cheat and toolchain
rationale.
